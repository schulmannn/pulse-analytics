## Часть 1 — Архитектура: вердикт

**Кодовая база сильно выше среднего для соло-продукта.** Фронт: один типизированный
API-клиент (fetch+Zod, ноль `any`), TanStack Query с разумными настройками, честная
токен-система дизайна, IG-слой (`useIgData` + Outlet context) — образцовый. Бэк: SQL
параметризован везде, ingest транзакционный и идемпотентный (receipts + payload-hash),
IG-токены AES-256-GCM at rest, OAuth state подписан HMAC с TTL, миграции под advisory lock,
анти-enumeration auth.

**Главный структурный долг:**
1. `server/index.js` — **1 908 строк**: роуты + IG-клиент + TG-прокси + email-шаблоны + CSP +
   статика. App не экспортируется → интеграционные тесты (supertest) невозможны, поэтому
   самые рискованные пути не покрыты.
2. Фронт: **TG-панели не получили IG-обработку** — деривации данных инлайном в рендере,
   без мемоизации, копипаст примитивов (`ChartSection` определён 5×, два пост-модала,
   `useDismiss` переизобретён 5×).
3. Систематический паттерн «без страховки»: падение процесса на unhandled rejection,
   ни одного таймаута на исходящие HTTP, коллектор ретраит «ядовитые» payload вечно.

## Часть 2 — Дизайн: что реально «выдаёт AI» (live-обход прода)

**Что уже выглядит «designed» (сохранять):** KPI-ledger на hairlines; блок «Инсайт»
(Итог→Доказательство→Что сделать); статус данных с mono-таймстампами; один синий;
сортируемый лидерборд; авто-инсайты с action-стрелками; warm-paper light. Направление
Refined Technical правильное — впечатление «сгенерировано» создают конкретные недоделки ниже.

### P0 — баги полировки (каждый мелкий, вместе решают всё)

1. **Сырой markdown в текстах постов**: `[Хлопок и шалфей](https://…)`, `__…__`, `**…**`
   видны как текст в Топ постов / Посты / модалке поста. Самый громкий «неотполировано»-сигнал.
2. **«обновлено обновлено сегодня»** — `freshness()` возвращает label уже с префиксом
   (`lib/freshness.ts:39`), потребители добавляют «обновлено» второй раз
   (`components/DataHealth.tsx:74`, `DashboardLayout.tsx:271,282`, `panels/Overview.tsx:78`).
3. **Двойной H1 «Настройки»** — топбар и тело страницы.
4. **Английский в графиках**: оси «18 May / 24 Jun 21:00 / 22 Mar» (лейблы приходят с
   graphs-API), серии `Views`/`Shares` рендерятся как есть (`TgAnalytics.tsx:347-348` —
   `viewSeries.name`), заголовок модалки разворота «Shares». Русский UI ↔ английские графики.
5. **Тултип графика**: позиционируется поверх шапки страницы и **залипает** после ухода
   курсора (наблюдал «24 May: 4 903 подписчиков» поверх хедера при скролле). LineChart/ChartTooltip.
6. **Y-подписи внутри плота** — «4.8k» лежит на линии (Рост подписчиков), «47»/«189»
   обрезаны/перекрыты (Скорость набора, модалка поста). Зарезервировать левый отступ под ось.
7. **Клик по любому месту графика открывает модалку разворота** (ExpandableChart оборачивает
   весь чарт) — конфликтует с чтением crosshair. Разворот — только по иконке ↗.
8. **Дыра в сетке «Аудитории»**: левая колонка кончается («Активность по часам»), правая
   продолжается двумя чартами — огромное чёрное поле. Выравнивать пары/flow.
9. **BarChart при n=2 — два гигантских столба** во всю ширину (IG Reels «Ср. время
   просмотра»). Нужен max-width столбца (~48px) + центрирование.
10. **IG-карточки**: пустое (белое) видео-превью; обложки разной высоты → рваные дыры
    между фото и текстом.
11. **Противоречие данных**: Обзор-инсайт «Просмотры выросли на 5.0% к прошлому периоду»,
    а таб «Сравнение» — «Недостаточно истории для сравнения» (graphs vs archive). Свести
    источники или честно подписать оба.
12. Ошибка «Не удалось загрузить · 500» — без кнопки «Повторить»; при полном отказе API
    аноним видит её вместо лендинга.

### P1 — chart-craft и язык компонентов

13. **Вертикальный градиент на столбиках** (светлый верх → тёмный низ) — сейчас главный
    «шаблонный» приём. Плоская заливка + hover-подсветка (как steep/linear).
14. **Бейдж-спам в Топ постов**: зелёный «▲ +N% к среднему…» под ВСЕМИ строками — сигнал
    обесценен (`TopPosts.tsx:74`). Показывать только реальным выбросам (топ-1/2 или >2σ).
15. **Радуга в ERV/ER** (Посты): окрашено почти каждое значение, красный на каждой строке =
    шум/тревога. Красить только вне нормального диапазона (PctTag-пороги).
16. **Язык кнопок**: пилюли (Войти/Экспорт) vs прямоугольники (Обновить в Mentions — большой
    синий rect, Сменить фото/Удалить, Подключить Instagram). По системе CTA = pill — свести.
17. **Грязно-оливковые подсветки строк в дарке** (Состав вовлечённости «Репосты», По форматам
    «Фото») — chart-2 orange tint на тёмном даёт болотный. Отдельные dark-значения tint.
18. **Rail-иконки без тултипов** + hover-оверлей сайдбара внезапный; «Развернуть» незаметен.
19. **«UID: 1 · Owner: 1»** в Настройках каналов — внутренности наружу; спрятать за админа.
20. Пустое «Период vs предыдущий» — голый центрированный текст вместо EmptyState-паттерна.
21. Модалка разворота бар-чарта не добавляет ничего (нет осей/значений/пресетов) — обогатить
    или не разворачивать бары.
22. Login/Register: форма в пустоте (вся левая половина экрана пустая); кнопки Google на
    проде **нет** (фича shipped — проверить `GOOGLE_CLIENT_ID` на Railway).
23. Скролл-перф: дважды поймал замороженный рендерер на тяжёлых страницах (Mentions,
    Analytics) — профилировать связку sticky-header + SVG (совпадает с FH2).

### Steep: что взять (адаптируя под Refined Technical, НЕ копируя тёмный монохром)

- **S1. Period pager** «‹ 30д ›» рядом с пресетами — листать окна назад/вперёд. Дёшево,
  делает интерфейс «инструментом». (У steep — нижний таймбар с Week/Month/…/Custom + пейджер.)
- **S2. Метрика как объект**: их metrics-catalog (имя + определение простым языком + owner
  + категория) и metric-деталка (одна метрика на весь экран + панель Explore: Breakdown /
  Filter / Compare / Targets). У нас уже есть тексты InfoTooltip и KpiDrillDown — эволюция
  «Разбора» в полноценную страницу метрики: определение, большой график, breakdown по
  источникам/форматам, compare с прошлым окном.
- **S3. Value labels на ключевых точках** (пик/последняя точка) прямо на графике, не только
  в hover — steep подписывает точки в отчётах.
- **S4. Отчёт-документ для F3**: steep Reports = notion-подобный документ (иконка, крупный
  заголовок, глобальный фильтр-бар: период/страна/валюта, чередование текстовых секций и
  метрик-карт). Отличная модель для наших «share/PDF-отчётов» вместо скриншота дашборда.
- **S5. Состояние в URL** (период/таб/сорт в query) — шарабельные ссылки; у steep всё в URL
  (`?grain=auto&period=1m&…`).
- Подтверждение нашего паттерна: их country-ledger «SE 1.8M | FR 1.79M | …» — ровно наш
  KPI-ledger. Это сильная сторона, развивать.
